# 真实订单批量录入工作台

## 目标

调度员可以把微信或 Excel 中复制出来的大段订单文本一次粘贴到 React Parser 页面，系统按订单行拆分，生成待确认草稿。草稿默认以紧凑表格展示，只有展开或编辑时才进入修改状态。

## 当前能力

- `POST /api/parser/text` 保持单条解析兼容。
- `POST /api/parser/text` 增加 `batch: true` 时返回 `drafts` 和 `count`。
- 支持客户名单独一行，并附加到后续订单原文中，避免客户信息丢失。
- 支持包车、送迎混合文本拆单。
- 支持多段路线文本，例如 `大阪-奈良-宇治-京都`。
- 支持 `绿1500`、`10座600` 这类无空格价格。
- 支持 `儿童座椅*2`、括号费用备注、`+7000` 等备注保留。
- 解析失败仍生成草稿，并保留 `raw_text`。

## React 页面交互

入口：`http://127.0.0.1:5173/#parser`

页面结构：

1. 批量文本输入区
2. 待确认订单表
3. 展开详情
4. 原地编辑
5. 批量确认入库

待确认表主列：

- 选择
- 编号
- 开始日期/时间
- 结束日期/时间
- 路线
- 类型
- 车型
- 价格
- 备注
- 操作

## 操作建议

1. 粘贴微信订单文本。
2. 点击“批量解析为草稿”。
3. 在待确认表勾选订单。
4. 对有偏差的订单点“展开”或“编辑”。
5. 点击“原地确认”批量入库。
6. 到 Orders 页面检查订单池。

## 边界

- 本轮不接 OpenAI API。
- 本轮不做聊天式 AI UI。
- 本轮不做复杂 NLP，只做规则拆单和字段容错。
- 一键排单按钮本轮保留入口提示，实际派车仍沿用 Dispatch 链路。
