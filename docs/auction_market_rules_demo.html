计算规则

基础路线价 KIX -> 大阪 = 500 由平台后台维护路线价格表，按城市、机场、公里数、常规耗时和历史成交价制定。
服务修正 车型 + 语言 + 时段 车型、语言、早晚班、节假日、人数和行李都会形成加减价。
保护起拍价 成本线 + 最低毛利 由燃油、人工、停车高速、空驶、平台保障费和最低毛利组成，不允许被竞价打穿。
成交推荐 价格不是唯一条件 系统结合履约、服务、响应、责任记录和报价合理性推荐，不默认最低价中标。
等级溢价 B 级旅行社 / 标准匹配 = +0 高等级旅行社优先匹配高等级车公司；高等级车公司可获得服务溢价，低等级组合进入观察或人工确认。

谁制定和控制规则

平台运营制定，财务和服务负责人共同审批

路线基准价、保护起拍价、急单加价、回程折扣必须有版本号、生效日期和审批人。普通调度员不能临时改保护价。

依据不是拍脑袋，而是成本和服务事实

参考历史订单、平均耗时、燃油停车高速、司机工时、空驶率、季节波动、投诉率、迟到风险和旅行社预算接受度。

规则可灰度，不可暗箱

后台记录每次规则改动；对车公司公开“可报价区间”和“评分影响因素”，但不公开保护起拍价和其他公司报价。

互评不是单向扣车公司

评分方 评价对象 主要内容
旅行社/旅客 车公司与司机 准时、车辆、礼貌、语言、沟通、是否按约执行。
车公司/司机 旅行社与旅客 资料是否准确、航班是否变更未通知、客人迟到、超行李、临时改路线。
平台 双方责任 根据定位、聊天、航班、照片、时间戳和调度记录做责任归因。

责任归因规则

车公司责任

未按时出车、车况不符、司机联系不上、无故取消、未及时上报异常。

旅行社/旅客责任

航班或人数行李变更未同步、客人迟到、临时加点绕路、联系人失联、订单信息错误。

不可抗力或平台责任

事故、道路封闭、极端天气、系统通知失败、平台错误派发。此类不自动扣车公司分。

客诉仲裁与保证金

场景 处置方式 资金与扣分
旅行社乱发单或不结算 平台核对订单记录、聊天确认、取消时间、客人是否真实存在；高风险旅行社进入预付或保证金模式。 冻结旅行社保证金；扣旅行社信用分；严重时暂停发布大厅订单。
车公司接单后迟到/晚点 核对司机定位、出发时间、交通异常、航班变化、平台通知时间；区分可抗辩迟到和责任迟到。 责任成立时扣车公司保证金或运费；扣履约分；造成旅行社实际损失时按规则赔付。
司机态度或服务问题 要求旅行社/旅客提交文字、录音、照片、聊天截图；车公司可提交司机说明和现场证据。 有效客诉扣服务分；严重服务事故可部分退款、赔付或限制司机继续接高等级订单。
双方都有责任 按责任比例处理，例如旅行社航班变更未通知 60%，车公司未及时确认 40%。 保证金和扣分按比例分摊，不做单边处罚。
保证金不是罚款池

保证金只用于已确认责任的赔付、未结算订单、恶意取消和严重服务事故。冻结、扣款、申诉、解冻都必须有记录。

赔付要有上限和证据

普通迟到按订单金额比例赔付；误机、严重客诉、替换车辆等重大损失进入人工仲裁，不允许无限扩大赔偿。

等级升降机制

对象 升级依据 降级依据
车公司 准时率高、客诉低、车辆语言匹配稳定、异常上报及时、合理报价。 无故取消、迟到不报备、车况不符、恶意低价、绕开平台交易。
旅行社 订单资料准确、变更及时、客人配合度好、确认和结账及时。 频繁错单、航班变更不通知、客人长期迟到、压价后追加要求、拖欠结算。
