# 老板汇报三屏演示说明

## 三个画面怎么摆

### 1. 左侧：司机端小程序

打开微信开发者工具：

- 项目目录：`miniapp`
- 页面：`pages/driver/index?driver_id=1`
- 关闭顶部绿色的“可视化”模式
- 点击“编译”

这块给老板看的是：司机收到订单以后，在手机上如何确认接单、出库、导航、拍照、完成订单、提交费用。

### 2. 中间：后台运营端

打开 React 后台：

- 调度页：`http://127.0.0.1:5173/#dispatch`
- 司机监控：`http://127.0.0.1:5173/#driver-monitor`
- 财务页：`http://127.0.0.1:5173/#finance`

建议演示时先停在 `Dispatch`，派车后切到 `Driver Monitor`，司机提交费用后切到 `Finance`。

这块给老板看的是：调度端如何派单、如何看到司机状态变化、财务如何看到司机费用。

### 3. 右侧：流程解释窗口

运行：

```bash
python scripts/demo_full_runtime_flow.py --driver-id 1
```

脚本会每一步暂停，按回车继续。

右侧窗口给老板看的不是代码，而是“当前系统正在模拟哪个真实业务动作”。

## 一键启动三屏演示

推荐运行：

```bash
python scripts/start_boss_demo.py
```

它会：

- 打开后台调度页
- 启动一个新的 PowerShell 窗口
- 在新窗口运行分步演示脚本
- 固定把订单派给 `driver_id=1`，方便微信开发者工具司机端直接看到

微信开发者工具本身如果已经打开，不需要重复打开，只需要点“编译”。

## 演示流程

### 第一步：订单解析

右侧脚本显示：

```text
后台：粘贴 4 条订单文本并自动解析
```

后台应该看到解析草稿或新订单进入待处理链路。

讲解重点：

- 调度员可以复制微信或 Excel 里的订单文本
- 系统先生成草稿，不直接入库
- 解析不准的地方可以人工修正

### 第二步：人工修正并确认订单

右侧脚本显示：

```text
后台：模拟解析有误，已人工修正日期、路线、车型和价格
```

讲解重点：

- 真实运营中解析不可能 100% 正确
- 所以系统设计成“解析草稿 + 人工确认”
- 确认后才进入正式订单池

### 第三步：批量派车

右侧脚本显示：

```text
后台：选择同一个司机和车辆，批量派 4 单
```

后台 `Dispatch` 页面应该看到订单状态变化。

讲解重点：

- 调度员可以一次选择多单
- 选择司机和车辆后批量派车
- 系统会生成 assignment
- 订单同步到司机端

### 第四步：司机确认接单

右侧脚本显示：

```text
司机端：确认收到 4 个订单
```

司机端小程序应该能看到订单。若没有看到，确认：

- 页面是 `pages/driver/index?driver_id=1`
- 或在“我的”里切换司机 ID 为 `1`

讲解重点：

- 司机前一天晚上或当天可以看到调度分配的订单
- 点击确认后，调度端能看到司机已确认

### 第五步：出库流程

右侧脚本显示：

```text
司机端：车辆检查、酒精测试、点呼出库
```

讲解重点：

- 司机不是直接开车出发
- 出库前需要确认车灯、刹车、车身损伤、驾照、乘务员证、酒精、睡眠
- 点呼出库后，车辆状态进入已出库

### 第六步：订单执行

脚本会依次模拟 4 单：

```text
开始第 1 单，到达上车点
接到客人并上传照片
送达并完成订单
```

讲解重点：

- 到达上车点要报备
- 接到客人要拍照留证
- 中途地点可以拍照
- 送达后拍照并完成订单
- 一单结束后进入下一单

### 第七步：车辆入库

右侧脚本显示：

```text
司机端：全部订单完成，车辆入库点呼
```

讲解重点：

- 全部订单完成后，司机回库
- 入库清扫、点检、酒精测试、点呼入库
- 车辆状态回到入库/归库

### 第八步：费用联动财务

右侧脚本显示：

```text
司机端：提交停车费报销和代收车费
后台财务：查看完成订单和司机费用待确认池
```

后台切到：

`http://127.0.0.1:5173/#finance`

讲解重点：

- 司机可以提交停车费、高速费、门票等垫付费用
- 司机可以提交代收车费、夜班费、座椅费等代收款
- 财务端看到待确认费用
- 司机端不显示订单销售价格

## 汇报话术

这套系统现在演示的是完整闭环：

```text
订单文本
→ 自动解析草稿
→ 人工确认订单
→ 调度批量派车
→ 司机收到订单
→ 司机确认接单
→ 出库检查和点呼
→ 到达、拍照、接客、送达
→ 订单完成
→ 车辆入库
→ 费用报备
→ 财务待确认
```

老板需要看到的重点不是某个按钮，而是：

- 调度端和司机端已经联动
- 司机动作会回写后台状态
- 费用会进入财务端
- 订单执行有照片和报备记录
- 系统已经从单纯订单表，进入了运营闭环

## 常见问题

### 小程序点 Tab 没反应

先关闭微信开发者工具顶部绿色的“可视化”模式，再点“编译”。

### 司机端看不到订单

确认脚本使用了：

```bash
python scripts/demo_full_runtime_flow.py --driver-id 1
```

然后确认小程序页面是：

```text
pages/driver/index?driver_id=1
```

### 后台没有变化

刷新对应页面：

- 派车：`#dispatch`
- 司机：`#driver-monitor`
- 财务：`#finance`

如果还没有变化，运行：

```bash
python scripts/health_check.py
```
