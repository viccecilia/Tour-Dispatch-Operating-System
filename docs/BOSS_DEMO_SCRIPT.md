# 老板演示脚本

## 演示前准备

```bash
python scripts/reset_demo_db.py
python backend/main.py
```

打开：

```text
http://127.0.0.1:8000/dashboard
```

## 1. 打开 dashboard

看哪里：页面顶部 KPI。

预期看到：

- 今日订单
- 今日已派车
- 今日执行中
- 今日已完成
- 待确认草稿
- 未派车订单

说明重点：系统能一眼看到当天运营压力和待处理事项。

## 2. 展示解析订单

点哪里：顶部“解析订单”区域。

操作：

1. 粘贴一条订单文本。
2. 点击解析。
3. 查看“待确认订单（草稿解析）”表格。

预期看到：

- 草稿进入待确认表。
- 表格只显示关键字段。
- 展开后能看到完整原始文本和解析详情。

## 3. 草稿确认成订单

点哪里：待确认订单表中的“确认”或顶部“原地确认”。

预期看到：

- 草稿状态更新。
- 订单进入可派单池。
- 原始文本仍被保留在备注/详情里。

## 4. 分配司机和车辆

点哪里：

1. 勾选待确认订单或未派车订单。
2. 在司机列表选择司机。
3. 在车辆列表选择车辆。
4. 点击确认分配。

预期看到：

- 订单从上方待处理池减少。
- 下方已分配订单池增加。
- 订单号带上车牌后四位和司机缩写。

## 5. 查看日历排程

点哪里：日历排程区域。

操作：

- 切换 `24h`、`7d`、`30d`。
- 使用日期筛选。
- 查看车辆纵向、时间横向的占用条。

预期看到：

- 已派车订单按车辆显示。
- 不同状态用颜色区分。
- 多日订单在对应日期周期内可见。

## 6. 模拟司机端执行

点哪里：司机端页面或司机 API 测试入口。

操作顺序：

1. 司机确认订单。
2. 出库。
3. 到达。
4. 开始服务。
5. 完成订单。
6. 归库。

预期看到：

- 每一步都会写入 `driver_reports`。
- `execution_status` 持续更新。
- 调度端和 dashboard 能看到状态变化。

## 7. 回到 dashboard

看哪里：KPI 和状态数字。

预期看到：

- 执行中、已完成、已归库等数字随操作变化。
- 未报备数量减少。

演示结论：

这个 MVP 已经覆盖“录单、确认、派车、日历、司机回传、运营看板”的主链路。
