# Driver UI Guidelines

## 定位
司机端是移动执行工具，不是后台管理系统。页面要帮助司机在 3 秒内判断：
- 我是谁、我现在是什么状态
- 当前跑哪一单
- 下一步点什么
- 怎么导航、怎么拍照、怎么报备

## 顶部 Driver Hero
- 固定在页面顶部。
- 必须显示司机姓名、在线状态、当前车辆、日期时间、当前状态。
- 状态颜色：
  - 休息 / 已入库：深灰
  - 未出库：蓝色
  - 已出库 / 执行中：绿色
- Hero 不随底部 Tab 切换而改变。

## 主动作
- 每个页面只突出一个主动作。
- 主动作采用底部固定大按钮。
- 常见主动作：
  - 确认接单
  - 点呼出库
  - 到达上车点
  - 拍照等待客人
  - 送达拍照
  - 点呼入库
- 其他按钮使用辅助样式。

## 当前订单
- 当前订单卡必须比其他卡片更突出。
- 优先显示时间、路线、状态、下一步。
- 客人电话、车辆、导航作为快速操作放在订单卡下方。
- 后续订单默认折叠。

## 地图
- 使用微信小程序 `map` 组件。
- marker 规则：
  - 当前位置：蓝色语义
  - 上车点：绿色语义
  - 终点：红色语义
- 没有坐标时，不阻塞司机操作，提供文字地址和 Google Maps URL 复制。

## 费用
- 垫付和代收分开。
- 状态文案使用司机能理解的说法：
  - 未提交
  - 已提交财务
  - 财务确认
  - 财务驳回
  - 仍在司机手中
- 司机端不显示订单销售价格。

## 视觉
- 卡片圆角保持 20rpx 左右。
- 主按钮高度不低于 88rpx。
- 移动端避免大段说明文字。
- 弱化后台 label，强调动作和状态。

## R055 Premium Polish
- 首页只保留司机真正关心的运营数和当前任务，不放调度后台指标。
- 出入库使用分组 Step Flow：车辆检查、证件确认、健康确认、入库确认。
- 任务地图页先显示当前订单，再显示大地图和导航按钮，后续订单折叠。
- 费用页默认展示“新增垫付 / 新增代收”两个入口，表单点击后展开。
- 我的页面按个人中心设计，头像、收入概览、证件提醒、提醒区和工作履历分层展示。
