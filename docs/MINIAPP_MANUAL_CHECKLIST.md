# 小程序真机验收清单

## 1. 首页 dashboard

检查项：首页统计是否显示。

操作步骤：

1. 打开微信开发者工具。
2. 导入 `miniapp/`。
3. 确认 `miniapp/utils/api.js` 的 `baseUrl` 指向后端。
4. 打开首页。

通过标准：今日订单、未派车、已派车、待确认草稿等指标能正常显示。

失败时排查：检查后端是否启动、端口是否一致、开发者工具是否勾选“不校验合法域名”。

## 2. parser 页面

检查项：文本是否可粘贴并生成草稿。

操作步骤：

1. 进入解析页面。
2. 粘贴中文订单文本。
3. 点击解析。

通过标准：生成草稿，原始文本保留，解析失败也不会丢文本。

失败时排查：检查 `/api/parser/text` 请求是否成功。

## 3. 草稿确认

检查项：草稿是否可确认生成订单。

操作步骤：

1. 打开草稿详情。
2. 修正日期、时间、路线、车型、价格。
3. 点击确认入库。

通过标准：草稿状态变为 confirmed，订单列表出现新订单。

失败时排查：检查必填字段是否为空，查看 `/api/parser/drafts/{id}/confirm` 返回。

## 4. 订单页

检查项：订单是否可新增、编辑、软删除。

操作步骤：

1. 新增订单。
2. 修改价格或备注。
3. 删除订单。

通过标准：新增后列表出现，编辑后数据更新，删除后默认列表不显示。

失败时排查：检查 `/api/orders` 的 POST、PUT、DELETE 请求。

## 5. 派车页

检查项：订单是否可分配和取消分配。

操作步骤：

1. 选择未派车订单。
2. 选择司机。
3. 选择车辆。
4. 点击派单。
5. 对已派订单执行取消。

通过标准：派单后写入 assignments，订单状态变为 assigned；取消后订单回到 unassigned。

失败时排查：检查司机车辆是否 available，检查是否存在时间冲突。

## 6. 日历页

检查项：日历是否可切换 day、week、month。

操作步骤：

1. 进入日历页。
2. 切换 24h、7d、30d。
3. 更改日期。

通过标准：已派订单按车辆和时间显示，颜色状态清楚。

失败时排查：检查 `/api/calendar/dispatch` 返回是否包含 items。

## 7. 司机页

检查项：司机执行状态是否可流转。

操作步骤：

1. 输入或选择 driver_id。
2. 查看自己的订单。
3. 依次点击确认、出库、到达、开始服务、完成、归库。

通过标准：每一步状态更新，并写入 driver_reports。

失败时排查：确认 driver_id 和 assignment 是否匹配。

## 8. API 请求错误提示

检查项：失败提示是否清楚。

操作步骤：断开后端或改错 API 地址后操作页面。

通过标准：页面不白屏，能提示请求失败。

失败时排查：检查小程序 console 和 network 面板。
