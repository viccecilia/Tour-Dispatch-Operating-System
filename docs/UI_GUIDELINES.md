# UI Guidelines

## 视觉原则

当前 MVP 采用调度 SaaS 风格：

- 信息密度适中，优先展示待处理事项。
- KPI、表格、派车、日历、司机状态使用同一套颜色和圆角。
- 不使用装饰性渐变大背景，保持运营工具的清晰和稳定。
- 表格默认展示文本，只有编辑状态出现输入框。

## 颜色规范

- Primary：`#2563eb`，用于主要按钮、选中状态、主 KPI。
- Success：`#16a34a`，用于成功、已完成、确认分配。
- Warning：`#f59e0b`，用于未派车、待处理、未结账提醒。
- Danger：`#ef4444`，用于失败、异常、危险操作。
- Info：`#0891b2`，用于执行中、到达等中间状态。
- Background：`#f6f8fb`。
- Card：`#ffffff`。
- Border：`#dfe7f2`。
- Text Primary：`#0f172a`。
- Text Secondary：`#64748b`。

## 状态 Badge

- `assigned`：蓝色。
- `confirmed`：浅蓝。
- `departed`：黄色。
- `arrived`：青色。
- `in_service`：紫色。
- `completed`：绿色。
- `returned`：深绿。
- `failed`：红色。
- `pending`：灰色。

## KPI 规范

- 卡片高度统一。
- 标题使用二级文本色。
- 数字使用大号粗体。
- 底部使用 4px 状态色条。
- dashboard 只放关键运营指标：今日订单、已派车、执行中、已完成、未派车、草稿、未报备。

## 按钮规范

- 主按钮：蓝色。
- 成功动作：绿色，例如确认分配。
- 警告动作：黄色，例如原地确认。
- 次要按钮：白底边框。
- 危险动作：红色浅底。

## 间距与容器

- 页面外边距：24-28px。
- Section 卡片圆角：12px。
- 控件圆角：9-10px。
- 卡片阴影：轻阴影，不做厚重浮层。
- 表格行高：48px 左右。

## 页面布局

- Dashboard：KPI 在顶部，下面是核心工作流。
- Parser：输入区可折叠，草稿以表格/卡片展示。
- Dispatch：待确认订单、司机、车辆、预览形成一条派单链路。
- Calendar：车辆纵向，时间或日期横向。
- Driver：突出当前订单和下一步动作，少输入、大按钮。
