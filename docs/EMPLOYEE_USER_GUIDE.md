# 员工试运行使用说明

## 登录

管理端试运行账号：

- 管理员：`admin / admin123`
- 调度员：`dispatcher / dispatcher123`

司机端本轮仍使用轻量身份参数：

- `driver_id`
- 或请求头 `X-Driver-Id`

`driver_demo` 是司机角色演示账号，不能登录 Admin Console。

## 调度员录单流程

1. 打开 React Admin Console：`http://127.0.0.1:5173/#parser`
2. 把微信或 Excel 中复制出来的大段订单文本粘贴到输入框。
3. 点击“批量解析为草稿”。
4. 在“待确认订单表”检查日期、时间、路线、车型、价格。
5. 有偏差的订单点“展开”或“编辑”修正。
6. 勾选订单后点“原地确认”。
7. 到 Orders 页面确认订单已进入订单池。

## 调度员派车流程

1. 打开 `Dispatch`。
2. 在“未派车订单池”搜索或多选订单。
3. 点击“接龙排序”，必要时用上移/下移调整顺序。
4. 在司机池选择司机。
5. 在车辆池选择车辆。
6. 查看车型匹配和分配预览。
7. 点击“批量派车”。
8. 派车后检查“已派车订单池”和“日历同步预览”。

## 司机执行流程

1. 司机端通过自己的 `driver_id` 查看任务。
2. 依次提交：
   - 确认订单
   - 出库
   - 到达
   - 开始服务
   - 完成订单
   - 归库
3. 每次提交后调度端 Dashboard 和 Driver Monitor 会看到状态变化。

## 常见失败提示

- 派车冲突：司机或车辆同一时间已有 active assignment。
- 草稿 failed：解析失败但原文已保留，可以人工编辑后确认。
- API offline：后端未启动或端口不对。
