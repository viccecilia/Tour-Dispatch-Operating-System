# TA-STAGE-07 - 整体测试、上线与试运营

## Stage Goal

把旅行社端从本地开发推进到 trial 环境，并完成 Web、小程序、API、权限、订单大厅和跨公司联动的整体测试。这个阶段要求可演示、可回滚、可内部试运营。

## Scope

### In

- 端到端测试。
- 权限和租户隔离测试。
- 小程序体验版上传。
- Web trial 部署。
- API trial 部署。
- 数据备份和回滚。
- 操作文档和培训清单。

### Out

- 不在本阶段新增大功能。
- 不在本阶段做正式商业计费。
- 不在本阶段切换生产环境。

## Primary Roles

- 总平台管理员。
- 旅行社管理者。
- 旅行社客服。
- 旅行社导游。
- 旅行社财务。
- 车公司管理者。

## Deliverables

- 测试用例矩阵。
- 权限测试报告。
- 端到端业务流报告。
- 部署脚本和回滚说明。
- 小程序体验版版本号记录。
- Git 标签和发布说明。

## Rounds

### TA-S07-R01 - 端到端测试矩阵

- Goal: 覆盖从公司创建到订单完成结算的主流程。
- Work: 编写总平台、旅行社 Web、客服、导游、财务、订单大厅、车公司接单的测试用例。
- Acceptance: 每个角色都有输入项、点击项、预期结果和实际结果。
- Validation: 形成一份整体测试分析文档。

### TA-S07-R02 - 权限和租户隔离测试

- Goal: 确认不同公司、不同角色不能越权。
- Work: 用两个样例公司和多个角色账号测试数据隔离。
- Acceptance: A 公司看不到 B 公司订单、导游、财务和大厅私有记录。
- Validation: 调用 API 和页面访问都返回正确权限结果。

### TA-S07-R03 - 高订单量与性能测试

- Goal: 验证每天 50 到 80 单、每月约 2000 单时可用。
- Work: 导入样例订单，测试订单列表、日历、筛选和大厅列表。
- Acceptance: 页面不卡死，搜索和筛选保持可操作。
- Validation: 记录主要页面加载时间和明显卡顿点。

### TA-S07-R04 - Trial 环境部署

- Goal: 将最新代码部署到 trial API 和 trial Web。
- Work: 备份数据库，上传代码，安装依赖，重启服务，部署前端静态文件。
- Acceptance: `https://api-trial.taxi-airport.jp/api/ping` 返回 pong，Web 后台可访问。
- Validation: 完成部署后执行健康检查脚本和浏览器验证。

### TA-S07-R05 - 小程序体验版上传

- Goal: 上传旅行社端和相关车公司端小程序体验版。
- Work: 使用微信开发者工具或 CLI 上传，记录版本号和备注。
- Acceptance: 体验者可通过稳定入口进入最新版本。
- Validation: iOS 和 Android 体验者分别打开并完成登录测试。

### TA-S07-R06 - 关键业务回归

- Goal: 确认上线版本没有破坏既有车公司端。
- Work: 回归车公司管理端、调度端、司机端、运行管理端和旅行社端。
- Acceptance: 端口权限没有互相污染，tab 和页面与角色一致。
- Validation: 用每个角色账号登录，截图记录主页面和核心操作。

### TA-S07-R07 - 数据备份、回滚与运维入口

- Goal: 出问题时能恢复。
- Work: 固化数据库备份、代码回滚、服务重启、自检和日志查看步骤。
- Acceptance: 管理账户可看到必要的系统维护入口，非管理员不可见。
- Validation: 演练一次备份和一次服务软重启。

### TA-S07-R08 - 试运营交付

- Goal: 形成可给内部人员使用的试运营包。
- Work: 输出账号清单、测试入口、已知限制、操作说明和反馈收集表。
- Acceptance: 用户无需开发者协助即可完成基本测试。
- Validation: 安排一次完整模拟：录单、分配导游、发布大厅、车公司接单、导游执行、财务结算。

## Stage Exit Criteria

- 旅行社端可以在 trial 环境完成完整主流程。
- Web、小程序、API、权限和订单大厅均通过测试。
- 有明确版本号、Git 标签、部署记录和回滚说明。

## Risks / Decisions

- 微信体验版入口有有效期，稳定入口需要走正式版本或长期体验管理。
- Trial 数据和本地数据必须明确同步方向，避免账号在本地可用但云端不可用。
- 上线前不应把订单大厅当成真实交易市场，需要保持内部测试标识。

## Next Stage Input

- 根据试运营反馈进入正式产品化阶段，重点补齐订单大厅规则、计费、合同、消息通知和多公司运营后台。
