# TA-STAGE-04 - 旅行社导游小程序端

## Stage Goal

实现导游端小程序。导游端参照车公司司机端，但业务是导游任务执行，不包含车辆出库、入库和司机财务。导游只看到自己被分配的任务、行程节点、联系信息和上报入口。

## Scope

### In

- 导游登录。
- 微信绑定和免登录。
- 今日任务。
- 一周任务日历。
- 任务详情。
- 执行节点上报。
- 异常和照片上传。
- 个人资料和证件资料上传。

### Out

- 不显示车公司车辆维护。
- 不显示司机出入库。
- 不显示公司利润和结算金额，除非权限明确允许。

## Primary Roles

- 旅行社导游。
- 旅行社客服和管理者可查看导游任务状态。

## Deliverables

- 导游首页。
- 导游任务页。
- 导游地图页。
- 导游资料页。
- 任务确认回执。
- 异常上报和文件上传。

## Rounds

### TA-S04-R01 - 导游端登录与微信绑定

- Goal: 导游首次使用账号登录并绑定微信，之后可自动进入。
- Work: 小程序登录、账号校验、微信 openid 绑定、免登录。
- Acceptance: 已绑定微信再次打开无需输入账号密码。
- Validation: 新导游账号首次登录后绑定微信，退出再进入自动登录。

### TA-S04-R02 - 导游首页与任务摘要

- Goal: 导游首页展示最关键的今日事项。
- Work: 展示今日任务数、待确认任务、异常通知和下一任务。
- Acceptance: 已确认任务不重复占用首页提醒。
- Validation: 导游确认任务后，首页待确认数量减少。

### TA-S04-R03 - 导游任务日历

- Goal: 导游能查看自己一周和一个月任务。
- Work: 默认一周，下拉切换一个月，点击日期显示任务列表。
- Acceptance: 只显示当前导游自己的任务。
- Validation: 分配给其他导游的订单不可见。

### TA-S04-R04 - 任务详情与行程节点

- Goal: 导游能清楚看到当前任务、后续任务和执行节点。
- Work: 显示集合时间、地点、客户、路线、备注和联系人。
- Acceptance: 当前任务展开，后续待执行任务折叠。
- Validation: 有多单任务时，只展开最近待执行或执行中的任务。

### TA-S04-R05 - 执行节点上报

- Goal: 导游按业务节点提交回执。
- Work: 增加确认任务、到达集合点、接到客人、开始行程、行程完成等按钮。
- Acceptance: 每次点击写入时间、位置和操作人。
- Validation: Web 端可看到导游节点回执。

### TA-S04-R06 - 异常、照片和证据上传

- Goal: 导游可提交延误、客人未到、行程变更、事故等异常。
- Work: 支持文字、照片、定位和订单关联。
- Acceptance: 异常提交后客服和管理者收到通知。
- Validation: 上传照片后 Web 后台可以查看。

### TA-S04-R07 - 导游个人资料维护

- Goal: 导游可上传和更新自己的联系方式与证件。
- Work: 手机号、WeChat、Line、WhatsApp、Kakao、证件、有效期、健康体检文件。
- Acceptance: 导游端可提交更新，Web 端可审核和查看。
- Validation: 导游更新 Line 后，Web 导游资料同步显示。

## Stage Exit Criteria

- 导游能登录、绑定微信、查看任务、确认任务、执行上报。
- 导游只能看到自己的任务和资料。
- 客服和管理者能看到导游回执。

## Risks / Decisions

- 导游联系客户信息可能敏感，需要按订单展示，不应长期暴露。
- 位置上报需要处理用户拒绝授权。
- 文件上传需要大小限制和后台权限控制。
