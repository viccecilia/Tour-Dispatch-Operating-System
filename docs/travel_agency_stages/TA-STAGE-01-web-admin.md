# TA-STAGE-01 - 旅行社 Web 管理端

## Stage Goal

实现旅行社公司的 Web 管理后台。公司主账户通过这个后台维护下属账户、导游资料、客户资料、基础订单配置和权限范围。这个后台是旅行社公司的主控台，不是总平台后台。

## Scope

### In

- 旅行社 Web 登录。
- 首次登录强制改密。
- 公司主账户创建、修改、停用、启用下属账户。
- 修改账号姓名、手机号、账号代码和角色。
- 导游资料维护。
- 客户与旅行社资源维护。
- 公司基础设置。

### Out

- 不在本阶段实现订单大厅交易。
- 不在本阶段实现导游小程序执行流。
- 不在本阶段做复杂财务报表。

## Primary Roles

- 旅行社管理者。
- 总平台管理员只负责创建公司主账户，不进入公司内部日常管理。

## Deliverables

- Web 管理端导航与页面结构。
- 公司资料页。
- 账号管理页。
- 导游资料页。
- 客户资料页。
- 权限检查与审计日志。

## Rounds

### TA-S01-R01 - Web 管理端框架与主题

- Goal: 给旅行社 Web 做独立视觉入口，避免和车公司后台混淆。
- Work: 建立旅行社管理端导航、顶部栏、角色显示和公司名显示。
- Acceptance: 登录后一眼能看出当前是旅行社后台和当前公司。
- Validation: 用 DAITORA 公司主账户打开 Web 后台检查页面标题和角色标识。

### TA-S01-R02 - 公司主账户资料与首次改密

- Goal: 主账户第一次登录必须修改密码。
- Work: 增加 must-change-password 检查、改密表单和登录后跳转规则。
- Acceptance: 初始密码登录后不能进入业务页面，必须先改密。
- Validation: 用手机号后 6 位登录，确认被拦截到改密页；改密后可进入后台。

### TA-S01-R03 - 下属账户管理

- Goal: 公司主账户可以维护本公司账户。
- Work: 实现新增、修改、停用、启用、重置密码、角色分配。
- Acceptance: 姓名和手机号可修改；停用后可启用；重置后密码恢复手机号后 6 位并要求 Web 改密。
- Validation: 新建客服、导游、财务账号并分别登录检查权限。

### TA-S01-R04 - 导游资料管理

- Goal: Web 端维护导游基础资料，供导游小程序和订单分配使用。
- Work: 增加姓名、手机号、语言、证件、健康体检、上传文件、备注等字段。
- Acceptance: 导游资料可以新增、修改、停用和查询。
- Validation: Web 修改导游手机号后，小程序账号登录规则同步更新。

### TA-S01-R05 - 客户与资源资料管理

- Goal: 旅行社维护自己的客户、合作方、常用目的地和订单来源。
- Work: 建立客户列表、旅行社来源、常用路线和备注字段。
- Acceptance: 客服录单时可选择已有客户，也可新增客户。
- Validation: 新增客户后在订单录入页下拉列表可见。

### TA-S01-R06 - 公司基础设置与审计日志

- Goal: 重要变更可追踪。
- Work: 记录账号、导游、客户、订单配置的新增、修改、停用、启用。
- Acceptance: 管理者能查看本公司操作日志。
- Validation: 修改一个账号手机号后，审计日志记录操作人、时间和字段变化。

## Stage Exit Criteria

- 旅行社公司主账户能完成账号和基础资料维护。
- 下属账户登录权限与角色一致。
- 所有资料限定在本公司租户内。

## Risks / Decisions

- 手机号修改会影响登录名和默认密码生成，需要提示管理者。
- 导游资料可能包含证件照片，文件访问必须做权限控制。
- 公司主账户不能删除自己，避免公司失去管理入口。
