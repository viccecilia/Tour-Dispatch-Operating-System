# TA-STAGE-02 - 旅行社客服端订单录入与确认

## Stage Goal

实现旅行社客服端的核心工作流：录入订单、解析订单、补全客户和来源、确认订单、查询筛选和批量处理。客服端相当于旅行社的订单入口。

## Scope

### In

- 客服 Web 和小程序入口。
- 订单手动录入。
- 粘贴文本解析。
- 未确认订单池。
- 订单补充与确认。
- 客户、来源、旅行社字段可选可新增。
- 查询、筛选、批量确认。

### Out

- 不在本阶段实现财务结算。
- 不在本阶段实现完整订单大厅交易。
- 不在本阶段实现车公司车辆派单。

## Primary Roles

- 旅行社客服。
- 旅行社管理者可查看和修改全部客服订单。

## Deliverables

- 客服订单首页。
- 手动新增订单页面。
- 批量粘贴解析页面。
- 未确认订单列表。
- 订单详情与编辑弹窗。
- 客户来源选择与快速新增控件。

## Rounds

### TA-S02-R01 - 客服工作台首页

- Goal: 客服登录后看到今日订单、未确认订单、待分配导游和待提交用车。
- Work: 设计客服首页卡片、列表和快速入口。
- Acceptance: 客服无需进入管理后台也能看到当前待处理事项。
- Validation: 用客服账号登录，只显示客服相关功能。

### TA-S02-R02 - 手动新增订单

- Goal: 客服可逐条录入旅行社订单。
- Work: 建立订单号、日期时间、路线、人数、车型需求、客户、来源、备注等输入项。
- Acceptance: 新增订单默认进入未确认状态。
- Validation: 新增一条 KIX 到大阪订单后，在未确认列表可见。

### TA-S02-R03 - 粘贴文本解析订单

- Goal: 支持从微信、旅行社文本、表格复制内容中解析订单。
- Work: 识别关西、KIX、ITM、大阪、京都、神户、接机、送机、包车、车型、价格和备注。
- Acceptance: 解析结果可编辑，错误字段高亮。
- Validation: 用真实样例文本解析 3 条订单，确认时间、路线、类型、车型能识别。

### TA-S02-R04 - 客户与来源选择控件

- Goal: 订单页面的旅行社或客户字段既能筛选选择，也能快速新增。
- Work: 增加可输入下拉菜单和加号新增功能。
- Acceptance: 输入不在列表中的内容后，点击加号可新增到公司客户列表。
- Validation: 新增一个客户后，下一条订单录入可以直接下拉选择。

### TA-S02-R05 - 订单确认流程

- Goal: 客服确认订单后才能进入后续导游分配和用车提交。
- Work: 建立未确认、已确认、需补充、已取消状态。
- Acceptance: 未确认订单在日历上可显示为未确认，但不能进入最终执行。
- Validation: 双击订单编辑后保存，状态从未确认变为已确认。

### TA-S02-R06 - 查询、筛选和批量操作

- Goal: 支持高订单量下快速找到订单。
- Work: 按日期、客户、来源、路线、订单状态、导游、用车状态筛选。
- Acceptance: 50 到 80 单每天的场景下列表可用，不被卡片占满。
- Validation: 导入 80 条样例订单后搜索和筛选可正常响应。

## Stage Exit Criteria

- 客服可以完成订单从录入到确认。
- 客户和来源资料可以边录单边维护。
- 未确认订单和已确认订单状态清晰。

## Risks / Decisions

- 文本解析不能覆盖所有格式，必须保留人工编辑确认。
- 订单号生成要避免与车公司订单号冲突。
- 客服不能看到不属于本公司的订单。
