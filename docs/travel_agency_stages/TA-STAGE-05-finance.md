# TA-STAGE-05 - 旅行社财务端

## Stage Goal

实现旅行社财务端。财务端负责客户应收、车公司应付、导游报销、订单利润、结算状态和 Excel 导出。财务权限必须和客服、导游权限分离。

## Scope

### In

- 财务首页。
- 客户应收。
- 车公司应付。
- 导游报销。
- 订单利润。
- 结算状态。
- Excel 导出。
- 财务附件查看。

### Out

- 不让导游看到利润。
- 不让客服默认看到工资、报销、利润。
- 不在本阶段处理真实支付通道。

## Primary Roles

- 旅行社财务。
- 旅行社管理者可查看全部财务。

## Deliverables

- 财务订单列表。
- 应收台账。
- 应付台账。
- 导游报销台账。
- 订单利润报表。
- 结算锁定和导出功能。

## Rounds

### TA-S05-R01 - 财务首页与权限隔离

- Goal: 财务账号只看财务相关页面，其他角色不能看到敏感金额。
- Work: 建立财务导航、权限守卫和金额字段可见规则。
- Acceptance: 导游和客服账号看不到利润和工资报销。
- Validation: 分别用客服、导游、财务账号打开同一订单，检查金额可见范围。

### TA-S05-R02 - 客户应收台账

- Goal: 管理客户需要收款的订单。
- Work: 按客户、订单、日期、金额、结算状态记录应收。
- Acceptance: 可筛选未收、部分收、已收。
- Validation: 完成订单后生成应收记录。

### TA-S05-R03 - 车公司应付台账

- Goal: 管理向车公司支付的用车费用。
- Work: 关联车公司用车请求、订单大厅成交订单和固定合作订单。
- Acceptance: 可区分固定车公司订单和订单大厅成交订单。
- Validation: 车公司确认完成后生成应付记录。

### TA-S05-R04 - 导游报销与垫付

- Goal: 记录导游在行程中的垫付和报销。
- Work: 导游提交费用、财务审核、附件查看、状态流转。
- Acceptance: 财务可批准、驳回、标记已支付。
- Validation: 导游提交一条停车费后，财务台账可见。

### TA-S05-R05 - 订单利润与结算状态

- Goal: 按订单查看收入、成本和利润。
- Work: 汇总客户应收、车公司应付、导游费用和其他成本。
- Acceptance: 已结算订单可以锁定，锁定后普通角色不能改金额。
- Validation: 修改成本后利润自动更新；锁定后不能再改。

### TA-S05-R06 - Excel 导出

- Goal: 财务和管理者可按时间段导出台账。
- Work: 支持订单、应收、应付、导游费用和利润报表导出。
- Acceptance: 导出的 Excel 包含筛选条件和生成时间。
- Validation: 按本月导出财务报表并打开检查列名。

## Stage Exit Criteria

- 财务账号能完成应收、应付、报销和利润查看。
- 非财务角色不能看到敏感财务字段。
- 财务表格支持筛选和导出。

## Risks / Decisions

- 财务锁定后需要管理员解锁机制。
- 订单大厅成交价可能与旅行社客户报价不同，必须分开记录。
- 附件文件必须和财务权限绑定。
