# TA-STAGE-00 - 平台基础与旅行社租户模型

## Stage Goal

建立旅行社端的基础边界：公司租户、主账户、下属账户、权限矩阵、订单主数据和与车公司平台的分离规则。这个阶段不追求完整 UI，而是先把后续所有端口共用的数据和权限规则定下来。

## Scope

### In

- 总平台给旅行社公司分配公司主账户。
- 公司主账户在自己公司内创建下属账户。
- 旅行社角色包含管理端、客服端、导游端、财务端。
- 账户统一使用 `公司代码-手机号` 作为登录名。
- 初始密码为手机号后 6 位。
- Web 首次登录必须修改密码。
- 小程序端首次绑定微信后可免登录，不强制改密码。
- 所有数据必须带公司租户边界。

### Out

- 不在本阶段实现订单大厅竞价细节。
- 不在本阶段实现复杂财务结算。
- 不在本阶段做车公司司机端业务修改。

## Primary Roles

- 总平台管理员：创建公司、分配公司主账户、控制平台级功能。
- 旅行社管理者：管理本公司账号、订单、导游、财务和订单大厅发布。
- 旅行社客服：录入订单、确认订单、分配导游和提交用车需求。
- 旅行社导游：查看并执行自己相关的任务。
- 旅行社财务：查看应收、应付、报销和结算。

## Deliverables

- 旅行社租户模型设计。
- 账号命名与初始密码规则。
- Web 首次改密规则。
- 小程序微信绑定免登录规则。
- 平台级权限与公司级权限分离表。
- 旅行社订单基础字段表。

## Rounds

### TA-S00-R01 - 旅行社端产品边界确认

- Goal: 明确旅行社端和车公司端不是同一个业务端口，只通过平台级订单和订单大厅联动。
- Work: 列出旅行社 Web、管理端、客服端、导游端、财务端的职责。
- Acceptance: 每个角色能说清楚能看什么、能点什么、不能看什么。
- Validation: 形成一张角色功能矩阵，确认没有把车公司司机、车辆维护、出入库逻辑误放到旅行社端。

### TA-S00-R02 - 公司租户与主账户模型

- Goal: 建立公司级数据隔离。
- Work: 定义 company、company_code、master_account、tenant_id 等字段。
- Acceptance: 所有旅行社账号、订单、导游、客户、财务记录都能关联到同一个公司。
- Validation: 用 DAITORA 作为样例公司，验证 `DAITORA-080xxxx` 账号规则。

### TA-S00-R03 - 下属账号与权限矩阵

- Goal: 公司主账户可以创建自己公司的下属账号。
- Work: 定义管理、客服、导游、财务四类角色权限。
- Acceptance: 账号创建时必须选择角色，角色决定 Web 和小程序可见 tab。
- Validation: 用同一账号分别测试 Web 和小程序权限一致。

### TA-S00-R04 - 登录、改密与微信绑定规则

- Goal: 避免手机号后 6 位密码长期存在风险。
- Work: Web 首次登录检查 `must_change_password`；小程序首次登录强制绑定微信。
- Acceptance: Web 首次登录后必须改密；小程序绑定微信后再次进入自动登录。
- Validation: 新建账号后分别测试 Web 与小程序登录路径。

### TA-S00-R05 - 旅行社订单基础模型

- Goal: 为客服录单、导游任务、用车请求、订单大厅和财务结算提供统一订单表。
- Work: 定义订单号、来源、客户、日期时间、行程、人数、车型需求、导游、用车状态、结算状态等字段。
- Acceptance: 未确认订单、已确认订单、已分配导游、已提交用车、已完成、已结算都能表达。
- Validation: 用 3 条关西机场、大阪、京都样例订单跑状态流转。

### TA-S00-R06 - 总平台与公司平台功能分离

- Goal: 抽取只属于总平台的功能，避免交给旅行社公司自己修改。
- Work: 区分总平台功能和公司内功能。
- Acceptance: 总平台保留公司注册审核、套餐、订单大厅平台规则、跨公司仲裁、全局日志。
- Validation: 公司主账户不能修改其他公司数据，也不能修改平台级规则。

## Stage Exit Criteria

- 有清晰的租户、账号、角色、登录、微信绑定和订单基础模型。
- 总平台与旅行社公司后台边界明确。
- 后续 Web、小程序、财务和订单大厅都能基于同一套权限继续开发。

## Risks / Decisions

- 公司代码必须唯一，且登录名区分大小写规则需要固定。
- 小程序免登录依赖微信 openid 或 unionid，必须绑定到账号。
- Web 强制改密不能影响小程序绑定后的免登录体验。
