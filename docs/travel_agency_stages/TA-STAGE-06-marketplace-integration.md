# TA-STAGE-06 - 订单大厅与跨公司接单

## Stage Goal

实现轻量级订单大厅。旅行社或车公司把暂时无法执行的订单发布到大厅，其他有能力的车公司可以接单。旅行社端的发布流程需要从配单界面选中订单，跳转订单大厅，填写起拍价和一口价后确认发布。

## Scope

### In

- 订单大厅发布入口。
- 起拍价和一口价。
- 发布备注。
- 大厅订单列表。
- 车公司接单或报价。
- 成交后状态写回。
- 发布方和接单方通知。
- 审计与取消规则。

### Out

- 不在本阶段实现复杂竞价排名算法。
- 不在本阶段接入真实支付担保。
- 不在本阶段做公开市场营销页。

## Primary Roles

- 旅行社管理者。
- 旅行社客服。
- 车公司管理者。
- 总平台管理员。

## Deliverables

- 发布到订单大厅页面。
- 大厅订单列表。
- 车公司接单页面。
- 成交状态写回。
- 取消和超时规则。
- 总平台订单大厅监管页。

## Rounds

### TA-S06-R01 - 发布入口与草稿

- Goal: 从旅行社配单界面选中订单后进入发布流程。
- Work: 增加“放入订单大厅”操作，带订单上下文跳转到大厅发布页。
- Acceptance: 发布页自动带出订单号、时间、路线、车型和原始价格。
- Validation: 选中一条订单后跳转发布页，订单信息不丢失。

### TA-S06-R02 - 起拍价和一口价发布

- Goal: 发布方明确价格条件。
- Work: 输入起拍价、一口价、备注和有效时间。
- Acceptance: 一口价不能低于起拍价；价格为空不能发布。
- Validation: 起拍价 1200、一口价 1500 的订单可发布；反向价格被拦截。

### TA-S06-R03 - 大厅订单列表

- Goal: 有权限的车公司看到可接订单。
- Work: 显示时间、路线、车型需求、价格、发布方规则和备注。
- Acceptance: 本公司自己发布的订单和可接订单状态区分。
- Validation: 发布后车公司端大厅列表出现该订单。

### TA-S06-R04 - 车公司接单与报价

- Goal: 车公司可以一口价接单或提交报价。
- Work: 实现接单、报价、撤回、等待确认状态。
- Acceptance: 一口价接单立即进入待确认或成交状态，报价需要发布方确认。
- Validation: 车公司点击一口价接单后，发布方收到通知。

### TA-S06-R05 - 成交写回旅行社订单

- Goal: 大厅成交后原订单状态同步更新。
- Work: 写入成交车公司、成交价、成交时间和用车状态。
- Acceptance: 原订单不再显示为未安排用车。
- Validation: 成交后旅行社订单详情显示接单车公司。

### TA-S06-R06 - 取消、超时与责任记录

- Goal: 避免订单大厅订单长期悬挂。
- Work: 设置发布有效期、取消规则、接单后取消记录和责任方。
- Acceptance: 超时自动下架；取消必须记录原因。
- Validation: 设置短有效期订单，超时后从大厅下架。

### TA-S06-R07 - 总平台监管与审计

- Goal: 总平台可以监管跨公司订单大厅。
- Work: 平台后台查看发布、报价、成交、取消、投诉和异常。
- Acceptance: 总平台可冻结异常订单大厅记录。
- Validation: 用平台管理员查看跨公司大厅流水。

## Stage Exit Criteria

- 旅行社可以把无法执行订单发布到大厅。
- 车公司可以接单或报价。
- 成交结果能写回原订单。
- 总平台可监管大厅记录。

## Risks / Decisions

- 订单大厅涉及跨公司数据，字段公开范围必须收敛。
- 成交后取消责任要提前定义。
- 后续如果加入真实支付担保，需要升级交易模型。
