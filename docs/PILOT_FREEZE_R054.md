# WX-DISPATCH-R054 内部试运营冻结说明

## 冻结范围

R054 之后进入内部试运营稳定版，默认冻结以下主链路：

- 订单解析：文本解析、草稿确认、订单入库。
- 订单管理：订单大表、状态、基础编辑。
- 派车工作台：未派车池、司机/车辆选择、批量派车、冲突提示。
- 日历视图：24h / 7d / 30d 派车查看。
- 司机端：今日任务、出入库、报备、位置、照片、费用。
- 财务台账：订单财务、司机垫付/代收、结算状态。
- 通知中心：未确认、未报备、未上传照片、未提交费用等提醒。
- 证据链：订单执行 timeline、照片、小票、报备记录。
- 资源管理：司机、车辆、旅行社基础维护和到期提醒。

## 冻结规则

- 不新增大功能。
- 不重构系统目录和主链路。
- 只允许修复阻断试运营的问题。
- 所有修改必须保留订单、派车、司机、财务、证据链数据。
- 每次发布前必须完成本文件的验证命令。

## 回归清单

1. Parser：粘贴真实订单文本，生成草稿，人工修正，确认入库。
2. Orders：订单进入订单大表，可搜索、查看状态、查看证据。
3. Dispatch：选择订单、司机、车辆，派车成功，冲突能提示。
4. Calendar：派车订单出现在日历。
5. Driver：司机能看到任务，确认、出库、到达、完成、归库。
6. Evidence：上传照片、小票后，调度端可查看 timeline。
7. Finance：订单价格、司机垫付、代收和结算状态可查看。
8. Notifications：关键提醒可生成、可标记已读。
9. Backup：可生成 SQLite 备份。
10. Restore：可从备份恢复后健康检查通过。

```bash
python scripts/verify_driver_api.py
cd frontend

```

## 长时间运行观察

内部试运营期间建议至少观察半天：

- 后端是否持续可访问。
- Dashboard 数字是否异常跳变。
- 司机位置刷新是否停止。
- 通知是否重复过多。
- SQLite 文件是否能正常备份。
- 前端页面是否出现白屏或控制台错误。

## 试运营进入标准

- 全链路验证通过。
- 备份恢复验证通过。
- 老板演示流程可完整走通。
- 至少一名调度员、司机、财务人工确认可试用。
