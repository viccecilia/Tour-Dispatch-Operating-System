# 真实派车工作台

## 入口

React Admin Console:

`http://127.0.0.1:5173/#dispatch`

## 页面结构

1. 未派车订单池
2. 司机池
3. 车辆池
4. 分配预览
5. 已派车订单池
6. 日历同步预览

## 核心操作

### 未派车池

- 支持订单搜索。
- 支持多选订单。
- 支持全选与清空。
- 支持选中订单手动上移/下移调整顺序。
- 支持按日期、开始时间、起点做接龙排序。

### 司机与车辆

- 司机支持按姓名、电话、语言搜索。
- 车辆支持按车牌、车型、绿牌标签搜索。
- 车辆卡片展示车牌、车型、座位、状态、绿牌/颜色/雪胎等标签。
- 选中车辆后，未派车池会提示所选订单车型是否匹配。

### 派车

- 选中订单、司机、车辆后点击“批量派车”。
- 派车成功后写入 `assignments`。
- 订单状态更新为 `assigned`。
- 已派车池、日历、dashboard、司机端查询同步刷新。
- 如果司机或车辆时间冲突，页面显示冲突提示。

### 已派车池

- 展示订单、司机、车辆、价格、执行状态、最新报备。
- 支持取消分配。
- 支持勾选已分配订单后重新分配给新的司机和车辆。

## 验证脚本

`scripts/verify_dispatch_workbench_api.py`

覆盖：

- 创建 30 条未派车订单。
- 接龙排序 API。
- 批量派车。
- assignments 写入。
- orders 状态变 assigned。
- 日历可见。
- 司机端可见。
- 冲突拦截。
- 取消分配。
- 重新分配。

## 边界

- 不做拖拽日历。
- 不做地图。
- 不做 WebSocket。
- 不做复杂司机排班系统。
- 司机和车辆状态仍使用当前轻量资源模型。
