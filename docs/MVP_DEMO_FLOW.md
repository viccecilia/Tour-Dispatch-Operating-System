# MVP 演示流程

建议演示顺序：

1. 打开首页 dashboard
   - 看今日订单、已派车、执行中、已完成、未报备、待确认草稿。
2. 进入解析页 parser
   - 粘贴一条中文订单。
   - 生成草稿。
   - 修改起点、终点、价格或旅行社。
   - 确认生成订单。
3. 进入订单页 orders
   - 查看新订单是否进入订单大表。
   - 用日期、旅行社、关键词筛选。
4. 进入派车页 dispatch
   - 选择未分配订单。
   - 选择司机和车辆。
   - 一键分配。
   - 查看已分配列表中的执行状态和最新报备。
5. 进入日历页 calendar
   - 查看 24h 视图。
   - 切到 7 日视图。
   - 切到本月视图。
   - 点击订单查看详情。
6. 进入司机页 driver
   - 输入对应 driver_id。
   - 查看自己的订单。
   - 依次点击确认订单、出库、到达、开始服务、完成订单、归库。
7. 回到首页 dashboard
   - 查看执行状态数字变化。

演示数据：

- 运行 `python scripts/demo_seed.py` 可生成今日订单、已派车订单、执行中订单、已完成订单、parser 草稿和 dashboard 数据。

边界说明：

- 不演示地图、轨迹、照片上传、微信正式登录、财务计算。
- 本 MVP 重点演示调度、派车、日历、司机执行状态回传。
