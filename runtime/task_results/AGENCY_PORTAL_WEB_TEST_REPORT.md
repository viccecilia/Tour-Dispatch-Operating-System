## 1. 功能清单

### 页面：旅行社下单入口

#### 板块：页面头部

- 标题：旅行社下单入口
- 未登录状态：不显示退出按钮
- 已登录状态：显示退出门户按钮

按钮：

- 退出门户
  - 清除本地 `wx_dispatch_agency_token`
  - 回到旅行社登录卡片
  - 清除当前页面订单查询缓存

#### 板块：旅行社登录卡片

控件：

- 旅行社下拉框
  - 数据来源：`GET /api/agency-portal/agencies`
  - 显示 portal 已启用的旅行社
  - 当前样例包含：东京旅运、关西地接、富士观光、日和假期、樱花旅行
- Portal code 输入框
  - 用户输入旅行社门户代码
  - 登录请求参数：`agency_id` + `portal_code`

按钮：

- 登录旅行社门户
  - 调用：`POST /api/agency-portal/login`
  - 成功后保存 `wx_dispatch_agency_token`
  - 成功后进入订单后台
  - 失败时显示后端错误，例如 `401 invalid_agency_credentials`

#### 板块：提交订单

字段：

- 开始日期
- 结束日期
- 开始时间
- 结束时间
- 起点
- 终点
- 类型
- 车型
- 客人姓名
- 联系方式
- 人数
- 报价 JPY
- 备注 / 费用备注

按钮：

- 提交订单
  - 调用：`POST /api/agency-portal/orders`
  - 请求头：`X-Agency-Token`
  - 后端写入 `orders`
  - 自动写入当前旅行社 `agency_id` / `agency_name`
  - 自动设置 `order_source = agency_portal`
  - 默认派车状态：`unassigned`
  - 默认结算状态：`pending`
  - 成功后重置表单
  - 成功后刷新我的订单列表

#### 板块：我的订单

表格列：

- 订单号
- 日期时间
- 路线
- 车型
- 报价
- 派车
- 结算

数据规则：

- 调用：`GET /api/agency-portal/orders`
- 请求头：`X-Agency-Token`
- 后端按 `tenant_id` + `agency_id` 过滤
- 只显示当前旅行社提交或归属的订单
- 不显示其他旅行社订单
- 不显示车公司内部司机、车辆、利润等管理字段

## 4. 发现的问题

### P3：退出门户后登录表单状态未清空

现象：

- 从已登录状态点击退出门户后，页面回到登录卡片，但 React 状态里仍保留上一次选择的旅行社和 portal code。
- 如果不刷新页面，登录按钮会保持可点击。
- 刷新后的真实新会话中，登录按钮禁用逻辑正常：未选择/未输入时禁用，填入 code 后启用。

影响：

- 不影响安全性，因为后端会拒绝非法登录。
- 但退出后保留 portal code 不利于共享设备上的隐私和体验。

建议：

- 点击退出门户时同时执行 `setAgencyId("")` 和 `setPortalCode("")`。
- 保持当前登录按钮禁用条件：`!agencyId || !portalCode || loginMutation.isPending`。

## 5. 结论

旅行社后台 Web 当前可本地使用，核心链路已经跑通：

- 旅行社选择
- Portal code 登录
- 错误登录拦截
- 提交订单
- 订单写入后端
- 我的订单刷新
- 状态展示
- 表单重置
- 退出门户

需要优先修复的点是登录按钮的前端禁用态。除此之外，当前页面是一个轻量版旅行社下单后台，尚未包含完整旅行社运营后台的账号管理、导游管理、客户管理、订单大厅、财务台账等页面；这些能力目前在新后端模块和文档中已有接口/模型基础，但还没有全部接入这个 `#agency-portal` Web 页面。
