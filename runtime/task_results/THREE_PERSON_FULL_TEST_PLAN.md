# 旅行社版本三人全流程测试计划

生成日期：2026-05-31

## 0. 测试目标

验证本版本新增的旅行社端能力是否能和车公司端、司机端、平台总控完整联动。

本轮测试覆盖：

- 旅行社 Web 端
- 旅行社小程序端
- 车公司 Web 管理端
- 车公司调度/前端相关页面
- 司机小程序端
- 平台总控端
- 订单大厅
- 跨旅行社、跨车公司数据边界
- 订单录入、发布、接单、派车、执行、结算闭环
- 公司注册、公司缩写代码、藤本 PDF、银行资料

不执行：

- 云端部署
- 生产数据库修改
- 删除真实数据
- Git push
- 小程序上传

以上动作需要单独确认后再执行。

## 3. 第一阶段：公司注册与基础资料

### 3.1 平台总控检查公司注册页面

操作人：测试员 A

步骤：

1. 登录平台总控。
2. 进入 `公司注册` 页面。
3. 新增一个车公司测试资料。
4. 新增一个旅行社测试资料。
5. 填写公司缩写代码、公司名、登记会社名、法人姓名、地址、联系人、电话、银行资料。
6. 上传藤本 PDF。
7. 上传许可文件 PDF。
8. 上传银行资料 PDF。
9. 保存草稿。
10. 改为提交审核。

预期：

- 公司缩写代码必填。
- 车公司代码进入 `tenants.slug`。
- 旅行社代码进入 `agencies.agency_code`。
- 藤本 PDF 上传后列表显示已上传。
- 缺少必填项时页面提示。

### 3.2 公司资料反查

操作人：测试员 A

步骤：

1. 搜索公司代码。
2. 按公司类型筛选：车公司。
3. 按公司类型筛选：旅行社。
4. 按审核状态筛选：草稿、已提交、已通过。

预期：

- 筛选结果正确。
- 旅行社和车公司资料不混淆。
- 不影响原车公司车辆、司机、调度页面。

## 4. 第二阶段：旅行社 Web 入单

### 4.1 机场接送批量入单

操作人：测试员 A

步骤：

1. 登录旅行社 `AGA`。
2. 进入批量入单。
3. 选择机场接送。
4. 粘贴多条机场接送文本。
5. 点击解析。
6. 检查日期、时间、航班号、起点、终点、客人、电话、人数、行李、车型、价格。
7. 修改一条解析错误的数据。
8. 保存订单。

必填项：

- 日期
- 时间
- 接送地点
- 联系方式
- 起拍价
- 一口价

预期：

- 多条文本变成多条订单。
- 解析后可编辑。
- 缺必填项不能发布。
- 订单只进入 AGA 的订单列表和日历。

### 4.2 包车批量入单

操作人：测试员 A

步骤：

1. 选择包车批量解析。
2. 粘贴多条包车文本。
3. 检查路线、日期、时间、车型、价格、客人名。
4. 上传标准行程 PDF。
5. 保存订单。

预期：

- 包车支持多条解析。
- PDF 属于每个行程，不放在发布大厅动作里。
- 跨日包车在旅行社日历条中横跨显示。

## 12. 第十阶段：变更与撤回

操作人：测试员 A、测试员 B

### 12.1 未接单撤回

步骤：

1. 旅行社发布一条新订单。
2. 不让车公司接单。
3. 旅行社点击撤回。

预期：

- 直接撤回成功。
- 大厅下架。
- 订单列表标记已撤回。
- 日历状态同步。
- 可重新发布，重新发布走新的大厅编号。

### 12.2 已接单修改

步骤：

1. 旅行社对已接单订单申请修改关键字段。
2. 车公司查看变更申请。
3. 车公司确认。

预期：

- 旅行社不能直接改关键字段。
- 车公司确认后才能修改。
- 修改记录保留。
- 订单不可直接撤回。

### 12.3 已接单取消

步骤：

1. 旅行社申请取消已接单订单。
2. 车公司确认。
3. 分别测试 24 小时以上、12 小时前、6 小时前规则。

预期：

- 24 小时以上且月度免费次数未超：免费取消。
- 每月最多 10 单免费取消。
- 12 小时前强制取消：收取 50%。
- 6 小时前未确认强制取消：收取 100%。
- 平台端记录费用规则。

## 17. 最终验收标准

本轮三人测试通过需要满足：

1. 三家旅行社各自订单隔离。
2. 三家车公司可以在大厅看到基础信息。
3. 未接单前敏感信息不泄露。
4. 接单后买方车公司可见完整资料。
5. 车公司已接订单可混合显示多个旅行社来源。
6. 派车后旅行社可见司机和车辆。
7. 司机端任务和汇报正常。
8. 订单列表、日历、跟踪、结算状态一致。
9. 平台总控可见全链路。
10. 旅行社 Web 和小程序数据同步。
11. 司机小程序与车公司 Web 状态同步。
12. 公司注册资料、缩写代码、藤本 PDF、银行资料可录入和查询。
