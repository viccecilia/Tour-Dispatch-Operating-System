# 轻量微信小程序调度平台推进蓝图（MVP版）

## 项目目标

打造一个轻量级微信小程序调度平台，核心目标：

- 快速录入订单
- 统一订单大表
- 批量派车
- 派车日历可视化
- 后续支持司机端执行闭环
- 后续支持车辆、人员、财务管理

---

# 核心推进策略

## 重要调整

第一阶段 MVP：

不要先做语音。

正确顺序：

1. 基础框架
2. 订单录入与订单大表
3. 派车核心
4. 派车日历
5. 订单解析增强（文本 / Excel / 语音）

原因：

订单表 + 派车 + 日历 才是调度平台主链路。

语音、定位、照片都属于增强能力，不应该阻塞主流程。

---

# 第一阶段 MVP（Round 1-5）

目标：

实现：

录入订单 -> 查看订单大表 -> 批量派车 -> 派车日历查看 -> 基础统计

# Round 1

## Round ID
WX-DISPATCH-R001

## Round Name
基础框架与权限骨架

## 本轮目标

搭建轻量调度平台的最小可运行框架，包括：

- 小程序端
- 后端 API
- 数据库
- 登录
- 角色权限
- 运营中台首页骨架

## 本轮任务

### 执行任务

- 建立小程序项目基础结构
- 建立后端 API 项目结构
- 建立数据库初始化脚本
- 创建基础表：
  - users
  - drivers
  - vehicles
  - agencies
  - orders
  - assignments
- 实现管理员登录
- 实现角色：
  - admin
  - dispatcher
  - driver
- 建立运营中台首页骨架
- 建立基础导航：
  - 首页
  - 订单
  - 派车
  - 日历
  - 车辆
  - 财务

### 测试任务

- 测试后端服务可启动
- 测试数据库可初始化
- 测试管理员可登录
- 测试登录后能进入首页
- 测试未登录不能访问业务页

### 协作验收任务

- 人工确认首页结构是否合理
- 人工确认导航结构是否清晰
- 人工确认角色是否满足未来扩展

## 允许修改

- 小程序基础页面
- 后端 API 基础目录
- 数据库初始化脚本
- 登录相关代码
- 首页骨架

## 禁止修改

- 不做订单复杂业务
- 不做语音
- 不做司机定位
- 不做财务计算
- 不做复杂权限系统

## 必须运行的验证

- 后端启动验证
- 数据库初始化验证
- 登录接口验证
- 小程序首页访问验证

## 完成定义

- 管理员可以登录
- 登录后能看到运营中台首页
- 数据库基础表存在
- 首页导航完整
- 无明显空白页、报错页

## 输出格式

- 修改了什么
- 每个任务状态
- 验证结果
- 协作验收结果
- 未完成/风险
- 下一轮建议
